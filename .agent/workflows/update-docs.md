# DOCUMENTATION.md 업데이트 워크플로우

프로젝트의 `App.tsx` 또는 핵심 로직을 수정한 후에는 반드시 아래 절차에 따라 `DOCUMENTATION.md` 파일을 최신화해야 합니다.

## 업데이트 절차
1.  **변경 사항 분석**: 이번 수정 작업에서 추가된 기능, 변경된 UI, 수정된 버그 등을 정리합니다.
2.  **문서 대조**: `DOCUMENTATION.md` 파일을 읽어 현재 기술된 내용 중 더 이상 유효하지 않거나 업데이트가 필요한 섹션을 찾습니다.
3.  **내용 갱신**:
    - `[최신 업데이트 사항]` 섹션에 최신 버전을 명시하고 작업 내용을 추가합니다.
    - `[핵심 페이지 및 기능]` 또는 `[데이터 모델]` 섹션에 변경된 세부 사항을 반영합니다.
    - `[완료 기준]` 체크리스트를 업데이트합니다.
4.  **파일 쓰기**: `write_to_file` 또는 `replace_file_content` 툴을 사용하여 `DOCUMENTATION.md`를 갱신합니다.

## 주의 사항
- 모든 설명은 **한국어**로 작성합니다.
- 사용자의 원래 의도와 현재 개발된 결과물 사이의 차이점을 명확히 기록합니다.
- 새로운 기능 추가 시 반드시 해당 기능의 목적과 사용 방법을 문서에 포함합니다.
