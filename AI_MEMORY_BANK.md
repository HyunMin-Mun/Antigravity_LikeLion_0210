# 🧠 AI Memory Bank - 프로젝트 맥락 저장소

이 문서는 AI 에이전트(Gemini, Claude, Cursor 등)가 프로젝트의 '지속적인 방향성'과 '설계 의도'를 잊지 않도록 하기 위한 딥 컨텍스트 저장소입니다. 새로운 세션이 시작될 때 이 문서를 가장 먼저 탐독하십시오.

---

## 🧭 1. 북극성 (The North Star)
- **개발 철학**: "코딩을 모르는 사용자도 AI의 제안만으로 프로젝트를 완벽히 통제할 수 있게 한다."
- **최종 사용자 경험**: 사용자는 대시보드를 보는 것만으로도 리스크를 인지하고, AI가 제안한 '승인' 버튼만 눌러서 문제를 해결합니다.

---

## 🛠️ 2. 아키텍처적 지향점 (Architectural Intent)

### 2.1 현재의 제약과 기회
- **모놀리식 구조**: 현재 `App.tsx` 하나에 모든 것이 담긴 것은 의도된 '빠른 반복(Iterative Speed)'의 결과입니다. 리팩토링 시에도 **함수형 프로그래밍** 스타일을 유지하고, 복잡한 컴넌트보다는 **명확한 데이터 가독성**을 우선하십시오.
- **Firebase 중심 실시간성**: 모든 데이터는 클라이언트 로컬 상태(`useState`)보다 **Firestore의 데이터 스냅샷**을 우선적인 소스로 삼아야 합니다.

### 2.2 디자인 시스템 페르소나
- **톤앤매너**: "애플(Apple)의 간결함과 엔터프라이즈의 신뢰감"
- **핵심 요소**: `rounded-3xl`의 적극적 사용, `indigo-600` 포인트 컬러, 그리고 `z-index`를 활용한 레이어링(Modals, Floating Panels).

## 🚦 4. 수정 시 주의사항 (Operational Guardrails)

1.  **UI 수정**: 단순히 기능이 된다고 끝이 아닙니다. `Lucide-react` 아이콘의 일관성을 유지하고, 버튼 클릭 시 `hover:scale-105`와 같은 미세한 인터랙션이 포함되어 있는지 확인하십시오.
2.  **데이터 무결성**: 업무(WorkItem) 생성 시 사용자의 권한(`role`)에 따라 접근 범위를 엄격하게 제한하십시오. (현재 매니저는 모든 업무 수정 가능, 멤버는 본인 배정 업무 중심)
3.  **한국어 톤**: AI 어시스턴트의 말투는 "분석적이고, 정중하며, 대안을 제시하는 프로페셔널한 비서"의 어조를 유지해야 합니다.
